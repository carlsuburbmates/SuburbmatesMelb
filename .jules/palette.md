## 2024-05-18 - Explicit ARIA Labels on Review Action Buttons
**Learning:** Icon+text buttons that include dynamic numbers (like "Helpful (12)") can be confusing or ambiguous to screen reader users if read exactly as-is. It's often necessary to provide a full sentence `aria-label` describing the action and current state (e.g., "Mark review by [Name] as helpful. Currently 12 people found this helpful."). Decorative icons inside the button must then be hidden from screen readers using `aria-hidden="true"`. Finally, interactive elements that use custom components or basic tags should explicitly define their focus outlines using Tailwind's `focus-visible` to ensure keyboard navigability.
**Action:** When creating icon+text buttons with dynamic counts, always wrap the element in an explicit `aria-label` providing full context. Hide the decorative icon with `aria-hidden="true"`. Apply `focus-visible:outline-none focus-visible:ring-2` (and appropriate ring colors/offsets) to ensure keyboard accessibility.
