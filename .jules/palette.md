## 2024-05-18 - Added ARIA labels to ImageGallery Component
**Learning:** Reusable media components often contain many icon-only controls (e.g. prev/next, zoom, close). When these components are deeply nested or act as modals, users relying on screen readers have difficulty navigating without explicit ARIA labels. Adding descriptive `aria-label`s to buttons and marking decorative inner icons with `aria-hidden="true"` is essential for equitable access. We must also denote selected state in thumbnails using `aria-current="true"`.
**Action:** Before writing new gallery or carousel components, verify that all directional and modal-control buttons have concise, descriptive labels and that state is communicated accessibly.

## 2024-05-18 - CI workflow variables context isolation
**Learning:** `e2e-serve.mjs` runs a Next.js `build` in the background, which relies on the environment variables to pre-evaluate API routes. Previously, we only added dummy variables like `RESEND_API_KEY` to the direct `npm run build` step in `ci.yml`. When `e2e-serve.mjs` executed its isolated sub-build, it crashed with missing variables like `Missing API key. Pass it to the constructor new Resend("re_123")` because environment step bindings in GitHub Actions do not persist across multiple discrete run blocks.
**Action:** Always inject required backend dummy API keys directly into the `E2E tests (serve)` step's `env:` block in `ci.yml` if the script launches a Next.js build.
