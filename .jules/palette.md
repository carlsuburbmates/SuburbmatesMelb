## 2025-02-14 - Accessible Mobile Hamburger Menu State and Footer Social Links
**Learning:** For mobile components (like the Hamburger menu toggle button), a single static `aria-label="Toggle menu"` is insufficient since it does not communicate the menu's state (open vs. closed) to screen readers. In addition, when hiding descriptive text for mobile designs, `hidden` removes the text from the accessibility tree; using responsive sizing utilities with `sr-only` keeps the text visually hidden but audibly present for screen reader users.
**Action:** When working on interactive layout toggles, always link the button to the corresponding menu container using `aria-controls`, and dynamically update `aria-expanded` and `aria-label` values (`Close menu` vs `Open menu`). When creating responsive text alternatives for icons, strictly prefer `sr-only sm:not-sr-only` over `hidden`.
