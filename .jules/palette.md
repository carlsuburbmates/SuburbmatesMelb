## 2024-03-24 - Accessibility and Semantic Fixes for Inline Forms
**Learning:** Purely client-side or "inline" UI elements like newsletter signup inputs often lack semantic `<form>` wrappers and accessible labels, leading to poor screen reader experiences and missing default browser behaviors (like submitting via the Enter key). Without `<form onSubmit={(e) => e.preventDefault()}>`, pressing Enter might accidentally reload the page if it's the only form on the screen or if it's inside another form structure. Additionally, placeholder text is not a substitute for an accessible label.
**Action:** Always wrap inputs in a `<form>` tag, even if submission is handled purely via JavaScript. Include a visually hidden `<label>` (e.g. `className="sr-only"`) connected via `id` and `htmlFor`, and ensure the submit button has `type="submit"`.
