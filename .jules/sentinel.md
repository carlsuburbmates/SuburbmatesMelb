## 2025-01-20 - PostgREST Filter Injection Vulnerability

**Vulnerability:** User input for search terms was directly interpolated into Supabase `.or()` filter strings in `src/app/api/creator/route.ts`. PostgREST uses special characters like `,` (comma) to separate conditions and `()` (parentheses) to group them within the `.or()` and `.and()` query string syntax. If an attacker injects these characters into the search string, they can manipulate the filter logic, potentially bypassing intended restrictions or causing query errors.
**Learning:** Supabase's JavaScript client constructs query strings under the hood for methods like `.or()`. String interpolation directly into these methods exposes the application to filter injection vulnerabilities because the underlying API (PostgREST) interprets specific characters as control characters. Aggressively stripping all non-alphanumeric characters (like `/[^a-zA-Z0-9\s]/g`) breaks internationalization.
**Prevention:** Always sanitize user input passed to Supabase's `.or()` and `.and()` string filters by specifically stripping or escaping the PostgREST control characters: `,` and `()`. The targeted regex `/[,()]/g` should be used instead of a restrictive allowlist regex to ensure international text and other valid characters are not corrupted.
