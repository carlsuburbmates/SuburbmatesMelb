## 2024-05-24 - [Contact Form XSS & Header Injection]
**Vulnerability:** User input from the contact form API (`name`, `email`, `subject`, `message`) was directly interpolated into the raw HTML payload of outbound support emails (`sendEmail`), causing an HTML injection (Stored XSS within support clients) vulnerability. Additionally, newlines in header fields like `subject` or `replyTo` could potentially cause email header/CRLF injection depending on the email provider.
**Learning:** The base `sendEmail` utility does not provide automatic sanitization. Any caller injecting user input into raw HTML templates must sanitize data explicitly. Furthermore, header fields should have newline characters removed, while HTML body content needs full HTML entity escaping.
**Prevention:** Created centralized sanitization utilities (`escapeHtml`, `stripNewlines`) in `src/lib/html-sanitizer.ts`. All API routes handling external text input destined for raw templates must apply these standard sanitization helpers.
