## 2024-03-04 - [Fix Contact Form HTML Injection]
**Vulnerability:** The `/api/contact` endpoint was vulnerable to HTML injection because it directly interpolated user input (`name`, `email`, `subject`, `message`) into the HTML body of the email. It was also vulnerable to Email Header Injection (CRLF Injection) because the `subject` was interpolated directly into the email subject header without stripping newlines.
**Learning:** This exposes the application to potentially rendering malicious scripts or content in the support team's email clients, as well as attackers potentially injecting secondary email headers (e.g. `Bcc:`, `Cc:`). The core issue was a lack of HTML encoding and newline stripping *before* the input was passed to the `sendEmail` function.
**Prevention:** I implemented an `escapeHtml` utility in `src/lib/html-sanitizer.ts` to sanitize all user inputs meant for the HTML body, replacing sensitive characters with their respective HTML entities. I also implemented a `stripNewlines` utility to strip out newline characters from the subject intended for the email header. I ensured that formatting (like replacing newlines with `<br>`) happened *after* sanitization.
