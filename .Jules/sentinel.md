## 2024-05-24 - [Contact API Vulnerability] Missing Sanitization in Email Output
**Vulnerability:** Contact Form API (`src/app/api/contact/route.ts`) inserted raw user inputs (name, subject, message) directly into HTML email templates and headers. This allowed for Cross-Site Scripting (XSS) via HTML tags in any field, and Email Header Injection via newline characters in the plain-text subject header.
**Learning:** Even though inputs were validated with Zod, they were not sanitized for their destination context (HTML vs Plain Text). The codebase lacked a centralized module for output sanitization, leading to the use of raw strings. Furthermore, applying HTML sanitization to plain-text fields (like subjects) can lead to double-encoding artifacts, meaning plain-text fields require a distinct sanitization strategy (stripping newlines) from HTML fields.
**Prevention:** Always apply context-specific output encoding. Created `src/lib/sanitization.ts` with `escapeHtml` (for HTML contexts) and `stripNewlines` (for plain-text/header contexts). Going forward, all variables injected into email templates or headers must use these sanitization utilities.
